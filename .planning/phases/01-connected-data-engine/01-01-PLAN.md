<task type="auto">
  <name>Task 1: Create calculation module with computed balance, budget spent, and surplus functions</name>
  <files>src/lib/db/calculations.ts, src/lib/types.ts</files>
  <action>
Create `src/lib/db/calculations.ts` with the following functions:

1. `computeAccountBalance(accountId: string): Promise<number>` — Returns SUM of all transaction amounts for that account. If no transactions exist, return 0.

2. `computeAccountBalances(userId: string): Promise<Map<string, number>>` — Returns a Map of accountId -> computed balance for all accounts belonging to user. Use a single `prisma.transaction.groupBy({ by: ['accountId'], _sum: { amount: true }, where: { userId, accountId: { not: null } } })` query for efficiency.

3. `computeBudgetSpent(userId: string, categoryId: string, month: number, year: number): Promise<number>` — Returns sum of absolute values of negative transactions in that category for the given month/year. Date range: first day of month to first day of next month.

4. `computeAllBudgetSpent(userId: string, month: number, year: number): Promise<Map<string, number>>` — Returns Map of categoryId -> spent amount for all categories in the month. Single groupBy query on transactions where amount < 0, grouped by categoryId.

5. `computeMonthlySurplus(userId: string, month: number, year: number): Promise<MonthlySummary>` — Returns `{ income, spending, surplus }` where income = sum of positive transactions, spending = absolute sum of negative transactions, surplus = income - spending.

Add to `src/lib/types.ts`:
```typescript
export interface MonthlySummary {
  income: number;
  spending: number;
  surplus: number;
}
```

Import prisma from `./prisma`. All functions are async and use Prisma queries. Use `groupBy` and `aggregate` for efficiency — avoid loading all transactions into memory.

IMPORTANT: Do NOT modify or remove the `Account.balance` field in the schema. It stays for backward compatibility. We just stop using it as the source of truth for display.
